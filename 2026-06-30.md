## Execution Log

### SARDINE blueprint locked

Full spec: [NOTES/SARDINE 🐟.md](NOTES/SARDINE%20🐟.md) · [design options](NOTES/SARDINE%20design%20options.md)

**SARDINE** = *Small Artificial RAM-restricted Deep Intelligent Neural Engine* — modular chess engine for Wio Terminal, maximizing Elo per byte under 192 KB RAM / ~508 KB flash.

#### Locked component choices

| Layer | Choice | Summary |
|-------|--------|---------|
| Runtime | **R-A** | Pure C engine core (Cfish-style); minimal C++ only for TFT/Serial if needed |
| Features | **F-B** | Pruned 704 inputs (impossible pawn ranks + mirrored king coords) |
| Evaluation | **E-E (E-C)** | Bucketed micro NNUE `768→16→1`, 8 output weight sets |
| Buckets | **B-C** | Balanced piece-count buckets (equalized training mass per bucket) |
| Policy v1 | **P-A** | Search-only; upgrade to **P-B** (history heuristics) later |
| Updates | **U-B → U-C** | Incremental add/sub accumulators; lazy updates when TT is live |
| Geometry | **K-A, K-B, K-C** | King mirroring + zero impossible weights + magnitude pruning |
| Quantization | **Q-A** | int8 weights, int16 accumulators, CReLU |
| Search | **S-B → S-C → S-D** | Quiescence → LMR+NMP → iterative deepening |
| Flash | **M-F2** | ~10% flash for weights; rest for search code + tables |
| RAM | **M-R1** | TT-dominant (128–160 KB) |
| Move order | **O-A → O-B** | MVV-LVA first; killer moves when depth > 4 |
| Training data | **D-B + D-D** | Lc0 games + bucket-stratified resampling |
| Training pipe | **T-B + T-D** | nnue-pytorch + SPSA post-hoc tuning |
| I/O | **I-B** | TFT + Serial; hardcoded FEN for now |

#### Design targets (Details)

| Parameter | Target |
|-----------|--------|
| Elo | **≥ 2000** (FIDE Kaggle bots ~2500 Elo at 5 MiB RAM) |
| Move time | Best move within **~1 s** (no hard time cap) |
| MCTS | Feasible (1–50 ms/eval on device); **deferred to v2** — v1 uses alpha-beta |
| Net architecture | **Separate nets** — NNUE for eval + distinct net for search guidance later |

#### Architecture flow

```
FEN → F-B encoder → NNUE accumulators (U-B) → E-E eval (B-C bucket)
                         ↑
              alpha-beta search (S-B→C→D) ← TT (M-R1) ← move order (O-A)
                         ↓
                    best move → I-B (TFT/Serial)
```

#### Custom hybrid (vs presets)

Closest preset: **Sardine FIDE** / **Sardine NNUE**, but custom mix:
- FIDE-style memory (**M-R1**) + micro bucketed NNUE (**E-E**) + v1 search ladder (**P-A**, **S-B**)
- Pure **R-A** C core (not M-R2 eval-dominant RAM split)
